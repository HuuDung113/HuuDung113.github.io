Create a New Post Lambda Function

In this step, we will create a new AWS Lambda function to handle new posts for text-to-speech conversion.

Steps to Create the Lambda Function

  1. Open the AWS Management Console and search for Lambda.

  2. Choose Create function.

  3. Select Author from scratch and configure the following:

    • Function name: PostReader_NewPost
    • Runtime: Python 3.12
    • Execution role: Use an existing role
    • Existing role: Lab-Lambda-Role

  4. Choose Create function.

Create success!

Lambda Function Code

import boto3
import os
import uuid

def lambda_handler(event, context):
    # Generate a unique ID for the new record
    recordId = str(uuid.uuid4())
    voice = event["voice"]
    text = event["text"]

    print(f'Generating new DynamoDB record with ID: {recordId}')
    print(f'Input Text: {text}')
    print(f'Selected Voice: {voice}')

    # Create a new record in the DynamoDB table
    dynamodb = boto3.resource('dynamodb')
    table = dynamodb.Table(os.environ['DB_TABLE_NAME'])
    table.put_item(
        Item={
            'id': recordId,
            'text': text,
            'voice': voice,
            'status': 'PROCESSING'
        }
    )

    # Send notification about the new post to SNS
    client = boto3.client('sns')
    client.publish(
        TopicArn=os.environ['SNS_TOPIC'],
        Message=recordId
    )

    return recordId

Code Explanation

  1. Import Libraries
import boto3
import os
import uuid
  • boto3: AWS SDK for Python to interact with AWS services.
  • os: Access environment variables.
  • uuid: Generate unique IDs.
  1. Generate Record ID
recordId = str(uuid.uuid4())
  • A unique recordId is created for each post using uuid.uuid4().
  1. Extract Input Parameters
voice = event["voice"]
text = event["text"]
  • The function expects an event containing voice and text.
  1. Log Information
print(f'Generating new DynamoDB record with ID: {recordId}')
print(f'Input Text: {text}')
print(f'Selected Voice: {voice}')
  • Logs are printed to CloudWatch for debugging.
  1. Save Data to DynamoDB
dynamodb = boto3.resource('dynamodb')
table = dynamodb.Table(os.environ['DB_TABLE_NAME'])
table.put_item(
    Item={
        'id': recordId,
        'text': text,
        'voice': voice,
        'status': 'PROCESSING'
    }
)
  • The put_item() method stores a new item with fields: id, text, voice, and status.
  1. Publish Message to SNS
client = boto3.client('sns')
client.publish(
    TopicArn=os.environ['SNS_TOPIC'],
    Message=recordId
)
  • The publish() method notifies the SNS topic with the recordId.
  1. Return Record ID
return recordId
  • The ID is returned as the response for tracking purposes.

Configure Environment Variables

  • Open the Configuration tab in the Lambda console.
  • Choose Environment variables > Edit > Add environment variable.

  • Key: SNS_TOPIC
    Value: <Paste the SNS Topic ARN>
  • Key: DB_TABLE_NAME
    Value: posts

  • Save the changes.

Update Timeout

  • Go to General configuration > Edit.

  • Set Timeout to 10 seconds.

  • Save the changes.

Test the Lambda Function

  1. Open the Test tab.
  2. Configure a new test event named Joanna with the following payload:
{
  "voice": "Joanna",
  "text": "This is working!"
}
  1. Save the event.
  2. Click Test.

Expected Output:

  • Execution result: succeeded
  • Log output: Displays the generated recordId and the input parameters.

The Lambda function is now ready to handle new posts and trigger subsequent text-to-speech processing.
